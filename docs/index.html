    Introduction

    Welcome to Hello-Books, a simple application that helps manage a library and its processes like stocking, tracking and renting books. With this application users are able to find and rent books.

    The API can be consumed using any programming language as it returns JSON for all API calls. You can view code examples in the dark area to the right.

    Features

    The API has the following features:

    Unauthenticated Users

    • Creating an account
    • Signing in as an existing user
    • Reset password

    Authenticated Users

    • View and update profile
    • Change password
    • View books
    • Borrow and return borrowed books
    • View borrowing history
    • Receive email notification when a surcharge is place for late return of borrowed book

    Admin

    • Add, edit and delete books
    • Add, edit and delete categories
    • Group a books by category
    • Edit membership types
    • View and read borrowing notifications

    Forgot Password

    This endpoint verifies the user for password reset. User can enter either their username or email to receive password reset link and token to change their password.

    HTTP Request

    POST api/v1/users/forgot-password

    onSuccess - Status code (200)

    onError - Status code (400, 404, 409)

    Request Body:

    {
    "entry": "solking",
}

    Success Response

    {
    "success": true,
    "message": "Reset message has been sent to your registered email."
}

    Error Response

    {
    "success": false,
    "error": "User with entry not found in our database"
}

    Request Required Fields

    • Authorization: NA
    Parameter Default Description
    entry username or email This field is required

    On success a token is created and sent to the user email to reset password.

    Reset Password

    This endpoint resets the user password

    HTTP Request

    POST api/v1/users/reset-password?token={token}

    onSuccess - Status code (200)

    onError - Status code (400, 404, 500)

    Request Body:

    {
   "password": "solking",
   "password_confirmation": "solking",
}

    Success Response

    {
    "success": true,
    "message": "Your password has been reset successfully, please login with your new credentials"
}

    Error Response

    {
    "success": false,
    "error": "Your password could not be reset, please try again later"
}

    Request Required Fields

    • Authorization: NA
    Parameter Default Description
    password null This field is required
    password_confirmation null This field is required
    On success a token is created and sent to the user email to reset password.

    Errors

    The Hello Books API uses the following error codes:

    Error Code Meaning
    400 Bad Request -- Bad request.
    401 Unauthorized -- Your Authentication token is wrong.
    403 Forbidden -- The endpoint requested is hidden for administrators only.
    404 Not Found -- The specified item could not be found.
    409 Conflict -- Request conflict.
    422 Unprocessable entity -- Server understands the request but cannot process it
    500 Internal Server Error -- We had a problem with our server. Try again later.
    503 Service Unavailable -- We're temporarily offline for maintenance. Please try again later.
